## プロダクト原則

設計判断に迷ったときの拠り所となる3つの原則。

1. **モチベーション・ファースト** — 数字で詰めず、伸びを称賛する。失点表示より「あと何点で目標」の前向きフレーミングを優先。
2. **3タップ以内** — 主要アクション（勉強開始、答案アップロード、成績確認）は3タップ以内で到達。子どもでも迷わない情報密度。
3. **AIは伴走者、判定者ではない** — AIは「弱点の発見」と「次の一手の提案」に徹し、最終判断はユーザーに委ねる。

### ターゲット別の配慮（年齢適応UI）

| 学齢 | UIモード | 主な配慮 |
| --- | --- | --- |
| 小学生 | キッズモード | 漢字にふりがな、アイコン主体、文章短め、ゲーミフィケーション強め |
| 中学生 | スタンダード | 内申/高校入試を意識。提出物・定期テスト管理を前面に |
| 高校生 | アドバンス | 模試・大学入試。志望校逆算アドバイス、情報密度高め |

## ① UI/UX設計と画面フロー

### 全体ナビゲーション構造

```
[ログイン / オンボーディング（進路目標・学年設定）]
        │
        ▼
┌──────────────────────────────────────────────┐
│  グローバルナビ（モバイル=下部タブ / PC=左サイドバー）│
│  🏠 ホーム  📊 成績分析  🤖 AI学習  👥 フレンド  ⚙ 設定 │
└──────────────────────────────────────────────┘
```

- **モバイル:** 下部固定タブバー（5項目）。中央の「AI学習」をコーラル色のFAB風に強調。
- **PC/タブレット:** 左サイドバー常時表示 + メインコンテンツ2カラム。

### 画面フロー（主要導線）

```
ホーム ──「答案を登録」──▶ 撮影/アップロード ──▶ OCR解析中 ──▶ 間違い確認・修正 ──▶ 弱点登録完了
  │                                                                              │
  │                                                                              ▼
  └──「テスト対策」──▶ 範囲・難易度指定 ──▶ AI問題生成 ──▶ 演習 ──▶ 自動採点 ──▶ 理解度更新
  │
  └──「勉強開始」──▶ タイマー（科目選択）──▶ 終了 ──▶ 記録保存 ──▶ ランキング反映
```

### 1. ダッシュボード（ホーム）

ユーザーが最初に見る「司令塔」。スクロール最小で今日やるべきことが分かる。

- **ヘッダー（Mainスカイブルー `#0EA5E9` グラデーション）:** あいさつ + 学年 + 今日の日付。右上に通知ベル（未読はコーラルバッジ）。
- **サマリーカード群（角丸16px・白背景・ソフトシャドウ）:**
  - 「今週の勉強時間」リング進捗（Success ミントで達成度）
  - 「次の提出物」最短期限カウントダウン（期限間近はコーラル）
  - 「目標達成度」直近テストの目標比ゲージ
- **AIひとことアドバイス:** その日の優先タスクを1行で（例:「数学の二次関数、昨日3問ミス。10分の弱点テストがおすすめ」）。タップで該当機能へ。
- **クイックアクション（3つの大ボタン）:** 「勉強開始」「答案を登録」「テスト対策」。
- **UIイメージ:** アイスブルー `#F0F9FF` の余白を広く取り、カードを浮かせる。情報は「今日・今週」に限定し、過去分析は成績タブへ送る。

### 2. 成績分析

「自分の弱点がわかる」のコア画面。

- **推移グラフ:** 科目別の折れ線（主線はスカイブルー）。**過去データの重ね合わせ**（前回/前年同時期）を半透明の比較線で表示。目標ラインを点線で重畳。
- **レーダーチャート:** 科目バランスの得意/不得意を一目で。平均点との差分を塗り分け。
- **目標管理:** 各テスト・各科目の目標点を設定 → 達成で Success ミントの紙吹雪アニメーション。
- **ドリルダウン:** 科目タップ → 単元別正答率 → 弱点単元へ「克服テスト」CTA。
- **UIイメージ:** グラフは余白多めで線を主役に。凡例はチップ型。達成/未達はミント/コーラルで即座に判別。

### 3. AI学習（AIテスト生成・答案分析・進路アドバイザー）

タブ内タブで3機能を集約。

- **答案分析:** カメラ起動 → 撮影 → OCRで設問・正誤を抽出 → ユーザーが正誤を最終確認（誤検出を修正可能）→ 間違えた問題を `WrongAnswers` に保存。
- **AIテスト生成:** 「範囲（単元/教科書ページ）・問題数・難易度」を指定 → LLMが練習問題を生成 → 演習 → 自動採点 → 理解度ゲージへ反映。
- **弱点克服小テスト:** 蓄積した間違いから自動編成。間隔反復（忘却曲線）で再出題タイミングを最適化。
- **進路アドバイザー:** 進路目標（高校入試/大学/専門/就職）と現在成績から逆算。「志望校まで内申あと2、英語の長文強化が最短」など具体的な次の一手を提示。
- **UIイメージ:** AI応答はチャット風カード。生成中はスカイブルーのスケルトン + やわらかいローディング。

### 4. フレンド

「友達と楽しく競える」を安全に。

- **承認制フレンド:** ID/QRで申請 → 相手承認で成立。未成年保護のため公開範囲は限定。
- **フレンド内ランキング:** 「勉強時間」「テスト点（共有許可者のみ）」「連続学習日数」をタブ切替。1位はコーラルの王冠。
- **称賛リアクション:** 競争が苦手な子向けに「いいね/がんばれ」スタンプ。順位より伸び率を称えるモードも。
- **UIイメージ:** ランキングはカードリスト、自分の行をスカイブルーでハイライト。点数共有は明示的オプトイン。

### 5. 設定

- プロフィール（学年・進路目標・UIモード）
- 通知設定（提出物リマインド、フレンド申請、ランキング更新）
- プライバシー（点数共有のオン/オフ、フレンド公開範囲）
- 保護者連携（保護者ビュー招待 — 任意）
- データエクスポート / 退会

## ② 技術スタック・システム構成

### 推奨スタック

| レイヤ | 技術 | 採用理由 |
| --- | --- | --- |
| フロント | **React 18 + Vite + TypeScript** | 高速HMR、型安全、エコシステム |
| UI | **Tailwind CSS** + Headless UI | デザイントークン管理が容易、角丸/配色の一貫性 |
| グラフ | **Recharts**（または Chart.js） | 折れ線・レーダーを宣言的に |
| 状態/データ取得 | **TanStack Query** + Zustand | サーバー状態とUI状態の分離 |
| BaaS | **Firebase**（Firestore + Authentication + Cloud Storage + Cloud Functions） | 認証・DB・ファイル・サーバー処理を一括。無料枠で開始でき、Hosting まで統一できる |
| OCR | **Google Cloud Vision API**（手書き対応の Document Text Detection） | 手書き答案の精度。Firebase と同じ Google Cloud で連携が容易。代替: Azure AI Vision |
| LLM | **Google Gemini API**（問題生成・進路アドバイス・答案要約） | 無料枠があり画像/PDFを直接読解できる。`gemini-2.5-flash` を使用 |
| ホスティング | **Firebase Hosting** | フロント・バック・ルールを `firebase deploy` で一括公開、CDN配信 |
| プッシュ通知 | **Firebase Cloud Messaging（FCM）** / PWA | 提出物リマインダー |

> PWA化（manifest + Service Worker）で「ホーム画面に追加」「オフライン下書き」「プッシュ通知（FCM）」に対応し、ネイティブ風の体験を低コストで実現。

**重要:** OCR / LLM の API キーはクライアントに置かず、必ず **Cloud Functions（サーバー側）** を経由して呼び出す。キーは `firebase functions:config` / Secret Manager に格納。クライアントは Firebase の IDトークンのみ保持。

### OCR（答案分析）フロー

```
1. クライアント: 答案を撮影 → 画像をリサイズ/圧縮 → Cloud Storage の /answers/{uid}/ にアップロード
2. クライアント: Cloud Function `analyzeAnswer` を呼ぶ（storage path + testResultId）
3. Cloud Function:
     a. Storage から画像取得
     b. Cloud Vision (DOCUMENT_TEXT_DETECTION) で手書きテキスト抽出
     c. Gemini にレイアウト+テキストを渡し「設問番号・解答・正誤推定」を構造化JSONで取得
     d. 結果を answerSheets / wrongAnswers に保存（status=draft）
4. クライアント: 抽出結果を表示し、ユーザーが正誤を最終確認・修正（誤検出対策）
5. 確定 → wrongAnswers を confirmed に。弱点克服テストの母集団へ
```

### LLM（問題生成・進路アドバイス）フロー

```
1. クライアント: 範囲・難易度・問題数を指定 → Cloud Function `generateQuiz`
2. Cloud Function:
     a. プロンプト構築（学年/科目/単元/難易度 + 出力スキーマ指定）
     b. Gemini API 呼び出し（responseSchema で JSON 強制）
     c. 生成問題を検証（必須フィールド・選択肢数・答えの整合）
     d. questions / quizzes に保存
3. 進路アドバイス: ユーザーの目標 + 直近成績サマリを集計 → Gemini → アドバイス文 + 推奨アクション配列
```

> **コスト/レイテンシ対策:** 生成結果はキャッシュ（同一範囲の再利用）、ストリーミング応答でUX改善、レート制限と1日あたり生成上限を設定。

## ③ データベース設計案

**Cloud Firestore（NoSQL）** を前提とする。下表は「論理データモデル」で、各テーブル＝**コレクション**、各行＝**ドキュメント**に対応する。リレーションは外部キーの代わりに**参照ID（例: `userId`, `subjectId`）**で表現する。

Firestore の方針:
- 各ユーザーデータのドキュメントには **`owner`（= `auth.uid`）フィールド**を持たせ、`firestore.rules` で本人のみ読み書き可能にする（RDBの行レベルセキュリティに相当）。
- 集計が多いデータ（勉強時間ランキング等）は、読み取り高速化のため**集計値を非正規化**して保持する（例: `users` に週間合計をミラー）。
- マスタ（`subjects` / `units`）は読み取り専用コレクションとして全員から参照可。

### 主要コレクションとリレーション

#### `users`（プロフィール — `auth.users` を拡張）
| カラム | 型 | 説明 |
| --- | --- | --- |
| id (docID) | string | Firebase Auth の `uid` と一致 |
| display_name | text | 表示名 |
| grade | text | 学年（例: 中2, 高1） |
| school_type | enum | 小/中/高 |
| ui_mode | enum | kids / standard / advanced |
| career_goal_id | uuid (FK) | → career_goals |
| share_scores | bool | ランキングでの点数共有可否 |
| created_at | timestamptz | |

#### `subjects`（科目マスタ）
`id`, `name`（数学/英語…）, `school_type`, `display_order`

#### `units`（単元マスタ）
`id`, `subject_id (FK→subjects)`, `name`（二次関数…）, `grade`

#### `test_results`（テスト成績）
| カラム | 型 | 説明 |
| --- | --- | --- |
| id (PK) | uuid | |
| user_id (FK→users) | uuid | |
| subject_id (FK→subjects) | uuid | |
| test_name | text | 中間/期末/模試名 |
| test_date | date | |
| score | numeric | 得点 |
| max_score | numeric | 満点 |
| average_score | numeric | 平均点（任意・比較用） |
| target_score | numeric | 目標点 |

> 推移グラフ = `user_id + subject_id` で `test_date` 順、重ね合わせ = 期間フィルタ、レーダー = 直近テストを科目横断集計。

#### `answer_sheets`（答案アップロード）
`id`, `user_id`, `test_result_id (FK)`, `storage_path`, `ocr_status`(processing/done/failed), `ocr_raw`(jsonb), `created_at`

#### `questions`（問題バンク — AI生成・OCR抽出共通）
| カラム | 型 | 説明 |
| --- | --- | --- |
| id (PK) | uuid | |
| unit_id (FK→units) | uuid | |
| source | enum | ai_generated / from_answer / curated |
| body | text | 問題文 |
| choices | jsonb | 選択肢（記述はnull） |
| correct_answer | text | |
| explanation | text | 解説 |
| difficulty | int | 1–5 |
| created_by | uuid | 生成元ユーザー（任意） |

#### `wrong_answers`（弱点ログ）
`id`, `user_id`, `question_id (FK)`, `answer_sheet_id (FK, nullable)`, `user_answer`, `status`(draft/confirmed/mastered), `last_reviewed_at`, `review_count` — 弱点克服テスト・間隔反復の母集団。

#### `quizzes` / `quiz_items`（演習セット）
`quizzes`: `id`, `user_id`, `type`(ai_test/weakness/self_check), `unit_id`, `score`, `created_at`
`quiz_items`: `id`, `quiz_id (FK)`, `question_id (FK)`, `user_answer`, `is_correct`

#### `study_logs`（勉強時間）
| カラム | 型 | 説明 |
| --- | --- | --- |
| id (PK) | uuid | |
| user_id (FK) | uuid | |
| subject_id (FK) | uuid | |
| started_at / ended_at | timestamptz | |
| duration_sec | int | タイマー実測 |
| note | text | |

> ランキング = 期間集計 `SUM(duration_sec)`。連続学習日数は日次集計から算出。

#### `assignments`（提出物・タスク）
`id`, `user_id`, `title`, `subject_id`, `due_date`, `source`(school/cram_school), `is_done`, `remind_at` — カレンダー/リマインダー。

#### `unit_progress`（理解度）
`id`, `user_id`, `unit_id (FK)`, `self_check_level`(1–5), `quiz_accuracy`, `updated_at` — 進捗ゲージ。

#### `career_goals`（進路目標）
`id`, `user_id`, `type`(high_school/university/vocational/employment), `target_name`, `required_metric`(jsonb: 偏差値/内申/評定など) — 逆算アドバイスの基準。

#### `friendships`（フレンド・承認制）
| カラム | 型 | 説明 |
| --- | --- | --- |
| id (PK) | uuid | |
| requester_id (FK→users) | uuid | 申請者 |
| addressee_id (FK→users) | uuid | 被申請者 |
| status | enum | pending / accepted / blocked |
| created_at / responded_at | timestamptz | |

> 一意制約 `UNIQUE(LEAST(requester,addressee), GREATEST(...))` で重複防止。ランキングは `accepted` の双方向集合に対して算出。

### リレーション要約

```
users 1───* test_results *───1 subjects 1───* units 1───* questions
users 1───* study_logs                         units 1───* unit_progress *───1 users
users 1───* assignments                        questions 1───* wrong_answers *───1 users
users 1───* answer_sheets 1───* wrong_answers  questions 1───* quiz_items *───1 quizzes 1───* users
users 1───1 career_goals
users *───* users  （friendships: requester / addressee の二重FK）
```

## ⑤ 開発ロードマップ（MVP）

「弱点がわかる」を最短で価値提供し、その後にAI・ソーシャルへ拡張する。

### Phase 0: 基盤（1–2週）
- Firebase プロジェクト作成、Authentication（メール/匿名）、Firestore/Storage セキュリティルール方針確定
- React + Vite + Tailwind デザイントークン実装、ナビ骨格、PWA設定

### Phase 1: MVP — 「記録と可視化」（3–4週） ⭐コア
- **成績の手入力 + 推移グラフ + 目標管理**（AI不要で即価値）
- **勉強時間タイマー + 記録**
- **提出物管理（カレンダー/リマインダー）**
- ダッシュボード統合
> MVPの目的: AI/OCRなしでも「成績と学習を一元管理できる」状態を最速で届け、習慣化フックを作る。

### Phase 2: AI弱点克服（4–6週）
- 答案アップロード + OCR（Cloud Vision）+ 正誤確認UI
- 弱点ログ → 弱点克服小テスト自動生成
- AIテスト生成（範囲指定）+ 自動採点 + 理解度ゲージ
- レーダーチャート分析

### Phase 3: 進路アドバイザー（3–4週）
- 進路目標設定 + 現成績からの逆算アドバイス（Gemini）
- 間隔反復による再出題最適化

### Phase 4: ソーシャル（3–4週）
- 承認制フレンド + フレンド内ランキング（勉強時間/点数/連続日数）
- 称賛リアクション、点数共有のオプトイン

### Phase 5: 拡張
- 保護者連携ビュー、キッズモード（ふりがな）、ネイティブ風プッシュ強化、A/Bでモチベ施策検証

### 優先度マトリクス（MoSCoW・MVP判断）

| Must（MVP） | Should | Could | Won't（初期） |
| --- | --- | --- | --- |
| 成績入力・推移グラフ・目標 | OCR答案分析 | 進路逆算アドバイス | リアルタイム対戦 |
| 勉強タイマー・記録 | AIテスト生成 | 保護者連携 | 動画教材配信 |
| 提出物リマインダー | レーダー分析 | 称賛スタンプ多様化 | 課金/サブスク |
| フレンド・ランキング（簡易） | 弱点克服テスト | キッズふりがな | |

## 付録

### デザイントークンまとめ

| 用途 | 色 | HEX | 比率 |
| --- | --- | --- | --- |
| Base（背景/余白） | アイスブルー | `#F0F9FF` | 60% |
| Main（ヘッダー/主ボタン/主線） | スカイブルー | `#0EA5E9` | 30% |
| Accent（ランキング/通知/重要） | コーラル | `#FF7E5F` | 10% |
| Success（達成/正解） | ミント | `#10B981` | スポット |

- **角丸:** 基準 `16px`（カード）、ヘッダー大カーブ `28px`、チップ/ボタンは pill。
- **フォント:** 本文 Noto Sans JP / 数値・英字 Plus Jakarta Sans。
- **影:** スカイブルー寄りのソフトシャドウでカードを浮遊させる。

### アクセシビリティ（子ども/全年齢配慮）
- コントラスト比 WCAG AA 以上。色のみに依存せずアイコン/ラベル併用（色覚多様性）。
- タップターゲット 44px 以上。キッズモードはふりがな・読み上げ対応を想定。
- 操作の取り消し可能性（誤タップ救済）、アニメーションは `prefers-reduced-motion` 尊重。

### セキュリティ / プライバシー（未成年データ）
- **セキュリティルール必須:** すべての個人データは Firestore/Storage ルールで `owner == auth.uid` に限定（`firestore.rules` / `storage.rules`）。
- **APIキー秘匿:** OCR/LLMキーは Cloud Functions 内のみ（Secret Manager）。クライアントに露出させない。
- **最小共有:** 点数共有・フレンド公開は明示的オプトイン。デフォルト非公開。
- **保護者保護:** 未成年規約・同意フローを想定。答案画像は `/answers/{uid}/` に本人限定保存（ルールで保護）。
- **生成物検証:** LLM出力はスキーマ検証・不適切表現フィルタを通してから保存・表示。
