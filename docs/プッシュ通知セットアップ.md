# プッシュ通知（FCM）セットアップ

提出物の期限が近づいたら、端末にプッシュ通知でお知らせする機能です。

```
設定タブ「🔔通知をオンにする」→ FCMトークンを fcmTokens に保存
   ↓（毎日18:00 JST）
Cloud Function remindAssignments が、期限が今日/明日の未完了提出物を持つ
ユーザーへ通知を送信（firebase-messaging-sw.js が受信して表示）
```

> 前提: Firebase は **Blazeプラン**（定期実行に Cloud Scheduler が必要）。

---

## 1. ウェブプッシュ証明書（VAPIDキー）を発行

1. Firebase コンソール → ⚙ プロジェクトの設定 → **Cloud Messaging** タブ
2. **「ウェブプッシュ証明書」** → **「鍵ペアを生成」** → 表示された鍵（`B…` で始まる文字列）をコピー

## 2. VAPIDキーを環境変数に設定

### ローカル（`.env.local`）
```
VITE_FIREBASE_VAPID_KEY=（コピーした鍵）
```

### GitHub（自動デプロイ用）
リポジトリ → Settings → Secrets and variables → Actions → New repository secret
- 名前: `VITE_FIREBASE_VAPID_KEY`
- 値: コピーした鍵

> ※ フロントは「ビルド時」にこの値を埋め込みます。GitHub Secret を登録したら、Actions タブで最新ワークフローを **Re-run**（または何かをpush）してフロントを再ビルドしてください。

## 3. 関数をデプロイ（定期実行の送信側）

```bash
cd functions && npm install && cd ..
firebase deploy --only functions
```
- 初回は **Cloud Scheduler / Pub/Sub API の有効化** を求められたら許可してください。
- `remindAssignments`（毎日18:00 JST）と既存の `generateQuiz` がデプロイされます。

## 4. 使い方

1. アプリ（できれば**ホーム画面に追加したPWA**）を開く → **設定タブ → 「🔔通知をオンにする」** → ブラウザの許可ダイアログで「許可」
2. 提出物タブで、期限が今日/明日の宿題を登録しておく
3. 毎日18時(JST)に、未完了の提出物があれば通知が届く

> 動作確認をすぐしたい場合は、コンソールの **Cloud Functions → remindAssignments → 「テスト/今すぐ実行」** や、Cloud Scheduler のジョブを手動実行できます。

---

## 補足
| 項目 | 内容 |
| --- | --- |
| 送信タイミング | 毎日 18:00 JST（`functions/src/index.ts` の cron を変更可） |
| 対象 | 期限が「今日」または「明日」の未完了提出物 |
| iOS | iOS 16.4+ で、**ホーム画面に追加したPWA**から通知をオンにした場合に対応 |
| トークン | `fcmTokens/{token}`（本人のみ）。無効トークンは送信時に自動削除 |
| SW | `firebase-messaging-sw.js` はビルド時に env から生成（値はリポジトリに直書きしない） |

### トラブルシューティング
| 症状 | 対処 |
| --- | --- |
| 「VAPIDキー未設定」と出る | 手順2のenv設定 → 再ビルド（ローカルは `npm run dev` 再起動） |
| 通知が来ない | 設定で通知を許可したか、`firebase deploy --only functions` 済みか、提出物の期限が今日/明日かを確認 |
| iOSで出ない | Safariでホーム画面に追加 → アプリから通知オン（ブラウザのままでは不可） |
