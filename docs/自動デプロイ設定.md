# GitHub Actions で自動デプロイ（CI/CD）

`main` ブランチにマージ（push）すると、GitHub Actions が自動で **ビルド → Firebase へデプロイ** します。
ワークフロー本体は [`.github/workflows/firebase-deploy.yml`](../.github/workflows/firebase-deploy.yml)。

```
PR を main にマージ ──▶ GitHub Actions 起動 ──▶ npm ci → npm run build
                                          ──▶ firebase deploy（Hosting + ルール）
                                          ──▶ https://seiseki-2a70c.web.app 更新
```

動かすには、GitHub に **7つのシークレット** を1回だけ登録します。

---

## 1. サービスアカウント鍵を作る（デプロイ認証用）

GitHub Actions は人間の代わりにデプロイするため、Googleの「サービスアカウント鍵（JSON）」が必要です。

1. Firebase コンソール → ⚙ プロジェクト設定 → **サービス アカウント** タブ
2. 「新しい秘密鍵の生成」→ JSON ファイルがダウンロードされる
3. そのファイルを**テキストエディタで開き、中身を全部コピー**（`{` から `}` まで丸ごと）

> このJSONはパスワード級の秘密です。リポジトリにコミットせず、GitHubのSecretsにだけ貼ります。

権限が足りずデプロイで失敗する場合は、Google Cloud コンソール → IAM で、そのサービスアカウントに
**Firebase Hosting 管理者 / Cloud Datastore オーナー / Firebase ルール管理者** 等のロールを付与してください。
（多くの場合 Firebase が自動付与する `firebase-adminsdk` アカウントでそのまま通ります）

## 2. GitHub にシークレットを登録

GitHub のリポジトリ → **Settings → Secrets and variables → Actions → New repository secret**
で、以下を1つずつ登録します（名前は完全一致させること）。

| シークレット名 | 値 |
| --- | --- |
| `FIREBASE_SERVICE_ACCOUNT` | 手順1でコピーしたJSON全体 |
| `VITE_FIREBASE_API_KEY` | `.env.local` と同じ値 |
| `VITE_FIREBASE_AUTH_DOMAIN` | `seiseki-2a70c.firebaseapp.com` |
| `VITE_FIREBASE_PROJECT_ID` | `seiseki-2a70c` |
| `VITE_FIREBASE_STORAGE_BUCKET` | コンソールに表示された値 |
| `VITE_FIREBASE_MESSAGING_SENDER_ID` | `.env.local` と同じ値 |
| `VITE_FIREBASE_APP_ID` | `.env.local` と同じ値 |

> `VITE_FIREBASE_*` はクライアントに埋め込まれる公開値ですが、CIのビルドで必要なのでまとめてSecretsに入れています。

## 3. 動かす

- これらを登録後、`main` に push / マージすると自動デプロイが走ります。
- すぐ試したい場合は GitHub → **Actions** タブ → 「Deploy to Firebase」→ **Run workflow** で手動実行できます。
- 進捗・ログは Actions タブで確認できます。失敗したらログの赤い行を見て、足りないシークレットや権限を直してください。

---

## 補足：なぜ `firebase init hosting:github` を使わないのか

`firebase init hosting:github` でも同等の設定が自動生成できますが、その方法は **Hosting のみ** をデプロイします。
本リポジトリのワークフローは Firestore / Storage の**セキュリティルールも一緒にデプロイ**するため、
サービスアカウント方式（`firebase deploy --only hosting,firestore:rules,storage`）を採用しています。
