# Firebase デプロイ手順

このアプリ（React + Vite）を **Firebase Hosting** に公開し、**Firestore / Authentication / Cloud Storage** を有効化する手順です。すべて無料枠（Spark プラン）で始められます。

---

## 全体像

```
ローカルPC                          Firebase（クラウド）
┌──────────────┐   firebase deploy   ┌────────────────────────────┐
│ npm run build │ ─────────────────▶ │ Hosting（dist/ を配信）     │
│  → dist/      │                    │ Firestore / Auth / Storage  │
└──────────────┘                    └────────────────────────────┘
```

---

## 1. Firebase プロジェクトを作る

1. [console.firebase.google.com](https://console.firebase.google.com) を開く
2. 「プロジェクトを追加」→ 名前（例: `seiseki`）を入力して作成
3. 左メニューから以下を有効化:
   - **Build → Authentication** → 「始める」→ ログイン方法（メール/パスワード、Google など）を有効化
   - **Build → Firestore Database** → 「データベースの作成」→ ロケーション選択（例: `asia-northeast1`＝東京）
   - **Build → Storage** → 「始める」

## 2. Web アプリを登録して設定値を取得

1. プロジェクト設定（⚙）→ 「マイアプリ」→ Web アイコン `</>` を選ぶ
2. アプリ名を入れて登録すると `firebaseConfig` が表示される
3. その値を `.env.local` に転記する:

```bash
cp .env.example .env.local
```

```
VITE_FIREBASE_API_KEY=...
VITE_FIREBASE_AUTH_DOMAIN=seiseki.firebaseapp.com
VITE_FIREBASE_PROJECT_ID=seiseki
VITE_FIREBASE_STORAGE_BUCKET=seiseki.appspot.com
VITE_FIREBASE_MESSAGING_SENDER_ID=...
VITE_FIREBASE_APP_ID=...
```

> `.env.local` は `.gitignore` 済みなので Git にはコミットされません。
> なお Firebase の Web 設定値（apiKey 等）は公開されても問題ない種類の識別子です。データの保護は本リポジトリの **セキュリティルール**（`firestore.rules` / `storage.rules`）で行います。

## 3. Firebase CLI を準備

```bash
npm install -g firebase-tools   # 初回のみ
firebase login                  # ブラウザが開き Google 認証
```

`.firebaserc` の `your-project-id` を実際のプロジェクトIDに書き換えます（または対話選択）:

```bash
firebase use --add              # 一覧から選んでエイリアス default を設定
```

## 4. ローカルで動作確認

```bash
npm install
npm run dev      # http://localhost:5173 で表示確認
```

## 5. 公開（デプロイ）

```bash
npm run deploy   # = npm run build（dist/生成）+ firebase deploy
```

完了すると公開URLが表示されます:

```
✔ Deploy complete!
Hosting URL: https://seiseki.web.app
```

`firebase.json` の設定で、Hosting（`dist/`）に加えて **Firestore ルール** と **Storage ルール** も同時にデプロイされます。

---

## セキュリティルールについて

未成年データを守るため、初期ルールは「ログイン必須＋本人のデータのみアクセス可」になっています。

- `firestore.rules` — 各コレクションを所有者（`owner == auth.uid`）に限定
- `storage.rules` — 答案画像は `/answers/{uid}/...` に本人のみ、画像・10MBまで

機能追加でコレクションを増やしたら、ルールも合わせて更新してください。

## トラブルシューティング

| 症状 | 対処 |
| --- | --- |
| `firebase: command not found` | `npm install -g firebase-tools` を実行 |
| デプロイで権限エラー | `firebase login` し直す／プロジェクトの権限を確認 |
| 公開後に画面が白い | `.env.local` の設定値が正しいか、`npm run build` が成功しているか確認 |
| Firestore で `permission-denied` | ログイン済みか、ドキュメントに `owner` フィールドがあるか確認 |

## （任意）GitHub への push で自動デプロイ
